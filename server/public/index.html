Overview

What is the Skills Proficiency API?

A groundbreaking open standard that enables organizations to consistently describe, exchange, and interpret skills proficiency information across systems.

Interoperable

Works across HCM, assessment, learning, talent, and skills intelligence platforms with zero friction.

Precise

Structured proficiency levels with ranks, scales, and evidence sources for accurate skill measurement.

Complementary

Works alongside JEDx, Open Badges, CLR, CASE CTDL, and other standards seamlessly.

Trusted Career Profile

Skills API Role in TCP

The Skills API is the service layer that powers skills data for portable career credentials. It doesn't replace TCP — it feeds it.

Skills API
TCP / CLR
1 Person completes assessment, course, or gains experience
2 Skills API creates verified assertions with evidence
3 Skills data is packaged into TCP or CLR credential
4 Person shares credential; systems validate via Skills API

Key Differences

1

Proficiency depth is this API's strength

OB3 says "you earned this badge." This API says "you have Python at level 4 of 5 on the SFIA scale, demonstrated in a production engineering context." The proficiencyScale, proficiencyLevel.rank, and proficiencyContext fields have no direct OB3 equivalent.

2

OB3 has cryptographic trust; this API has none

OB3 credentials are signed with EdDSA or JWT and can be independently verified. This API relies on whoever runs the endpoint being trustworthy — the source field is informational only.

3

OB3 is credential-centric; this API is query-centric

OB3 produces portable documents a person carries in a wallet. This API is a service you call to look up skills — it doesn't produce portable artifacts.

4

They're complementary, not competing

OB3 issues verifiable credentials proving achievements/skills. This API aggregates and queries those credentials, adding proficiency-scale normalization across frameworks. The context-v1.json already extends the OB3 JSON-LD context, confirming this layered design.

Where This API Fills OB3 Gaps

  • Cross-framework proficiency comparison — OB3 has no way to say "this person is level 3 on scale X." This API normalizes proficiency across CEDS, CASE, ASN rubrics.
  • Multi-target queries — OB3 is person-centric. This API can query skills for a Job, JobRole, Framework, or LearningProgram, enabling matching use cases.
  • HR/workforce context — The targetType enum and performanceContext are HR-domain concepts absent from OB3.

Where OB3 Fills This API's Gaps

  • Trust/verification — No cryptographic proof in this API
  • Portability — OB3 credentials live in digital wallets; this API requires network access to the service
  • Revocation — OB3 has status lists and expiration; this API has no validity lifecycle
  • Rich evidence — OB3 can embed or link artifacts; this API only has opaque context URIs
